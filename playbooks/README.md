# 🛠️ SOC Incident Response Playbooks

This folder contains structured **incident response playbooks** designed
for common security incidents handled in a Security Operations Center
(SOC).

These playbooks represent how I approach **real-world alert
investigation and response**, following industry-standard methodologies.

------------------------------------------------------------------------

# 📌 Purpose

The goal of these playbooks is to:

-   Provide a repeatable investigation process
-   Standardize incident response workflows
-   Improve triage efficiency
-   Demonstrate practical SOC skills

------------------------------------------------------------------------

# 📂 Playbooks Included

-   Phishing Email
-   QR Code Phishing (Quishing)
-   Brute Force Attack
-   Password Spraying
-   Impossible Travel
-   MFA Fatigue Attack
-   Session Token Theft
-   Suspicious PowerShell Execution
-   Malicious File Download/Execution
-   Ransomware (Pre-Encryption Stage)

------------------------------------------------------------------------

# 🔍 What Each Playbook Contains

Each playbook follows a consistent structure:

-   Objective
-   Scope
-   Threat Description
-   Severity Classification
-   Escalation Criteria
-   Detection Sources
-   Investigation Steps
-   Detection Queries (Splunk / KQL)
-   Containment Actions
-   Eradication Steps
-   Recovery Process
-   MITRE ATT&CK Mapping

------------------------------------------------------------------------

# 🧠 SOC Skills Demonstrated

-   Alert triage and prioritization
-   Log and telemetry analysis
-   Threat behavior identification
-   Incident response lifecycle execution
-   Decision making and escalation

------------------------------------------------------------------------

# ⚙️ Framework Alignment

These playbooks are aligned with:

-   NIST SP 800-61 (Incident Response Guide)
-   MITRE ATT&CK Framework
-   SANS Incident Handling

------------------------------------------------------------------------

# 🚀 Why This Matters

These playbooks are not theoretical.

They reflect:

-   How a SOC analyst actually investigates alerts
-   How decisions are made under pressure
-   How incidents are handled end-to-end
